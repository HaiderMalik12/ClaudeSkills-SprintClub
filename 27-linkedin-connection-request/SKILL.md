# Skill: LinkedIn Connection Request Writer

## What This Skill Does
Generates one ready-to-send LinkedIn connection request message using Jay Abraham's Preeminence approach. Give it a prospect — get a message. No back-and-forth, no full sequence. Just the connection request.

## Inputs Required
Provide any combination of:
1. **Prospect name**
2. **Title and company**
3. **One specific thing about their work** — book, framework, post, methodology, company they built

That's it. If a LinkedIn URL or profile is provided, use it to personalise further.

---

## The Preeminence Formula

**Lead with THEIR work → connect as a PEER → no ask**

```
[Their specific work/idea + how it impacted you or your clients] + [Peer connection intent] + [No pitch]
```

**The 5 rules — always apply all 5:**
1. Reference something SPECIFIC they created (book title, framework name, company, methodology)
2. Show it shaped YOUR thinking or client work — not flattery, evidence
3. Peer-to-peer language — "fellow," "aligned," "overlap," "resonates"
4. Zero pitch, zero ask, zero CTA
5. Under 300 characters — punchy, no filler

**The test before sending:**
> "Would this message feel genuine even if they never bought anything?"
> If yes — output it. If no — rewrite it.

---

## What to Avoid

| ❌ Don't | ✅ Do instead |
|---------|-------------|
| "I help companies like yours..." | Reference their specific work |
| "I'd love to learn more about you" | Show you already know their work |
| "Would you be open to a quick call?" | End with connection intent only |
| "Great content!" | Name the specific piece |
| "I wanted to reach out because..." | Just start with their name |

---

## Output Format

Deliver exactly this — nothing more:

**Connection Request Message:**
> "[Message here — under 300 characters, ready to copy-paste]"

**Why this works:**
- [One line on the specific reference used]
- [One line on the peer framing]

## When to Use This Skill
- You have a prospect name and want one message fast
- You're working through a prospect list and need consistent preeminence-based messages
- You want to check if a message you wrote passes the preeminence test
